During my time as a support engineer I have used multiple Call Management systems, These have been as simple as no call system just using emails,  which in my experience does not enable a comprehensive method of transferring calls.  Through to Complex modern Web based Call management systems such as My Oracle Support (MOS).   Not forgetting primitive Disk Operating System (DOS) based system such as Servasure to name but a few.  The point is I am able to learn new technologies as and when required.   Another common factor of my support roles is I am comfortable speaking to Non-Technical users through to Technically Advanced Developers and usually translating between the different technical Levels.

At Indigo 4 Systems Ltd (2000  -  2004),  my role was as a Support   Programmer this meant that for two weeks I would code (FoxPro,  C# or VB) once I completed a module I would test a colleagues while they tested mine.   However,  the impact was greater in the Support element because while doing support for the product then we had access to the source code which gave context to a totally nonsensical problem.   This context meant that we could ask the right questions to identify the actual problem and ultimately reach a quick solution.

At Torex Retail Systems Ltd (2008  -  2012),  In comparison this was a purely Support role where access to source code wasn't given,  which made resolving issues longer.   Because in the event that the source code was required,  it would be necessary to contact Development who once they got round to your call would answer the question (in a fashion).    This employment required that we would have to learn approx 3 systems to support these sytems which included Electronic Point Of Sale (EPOS) &  Back Office (BO) Systems which were both console & web based applications.  Torex was absorbed into Micros Retail Systems (2012  -  2014), finally Oracle Retail UK (2012  -  2022).

The Oracle phase of my Software support Experience added a further dimension this was to change the context from supporting UK Retailers to supporting global businesses,  Introducing working methodologies such as chasing the sun,  which means that as you go off shift then the call is passed onto someone who is coming on shift usually in a different timezone.  So technically a call could be worked 24 / 7.  This phase of the employment also saw me elevated to Oracles equivelant of a 3rd Line Support Engineer.
