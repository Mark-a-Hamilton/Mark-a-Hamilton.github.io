On no account would I claim to be a 'Guru' of any web technology,  However,  having said that I have wrote this entire site from scratch using online resources such as bootswatch,  JQuery,  Bootstrap, etc.   combined with my knowledge of Web technologies & ability to research then apply what I need as and when required.   As a result I feel that I have proved that I have the abillty to move into a web development role.

Note:  As HTML5 & CSS3 have continually evolving specifications and as a result will always need researching,  this isn't a problem for me.

The prime objective of HTML is to provide the page content and structure,  these are not particularly complicated but having said that HTML5 is becoming more powerful.   For example the best addition in my opinion to HTML5 is the introduction of 'Event Listeners' such as 'OnClick'.   These enable a routine to be triggered based on a single event.   Not forgetting that Javascript (JS) scripts usually has the event listener wrote into the script so these JS scripts can be simplified and made more efficient as the JS event listener is effectively redundant.

This is another evolving technology and serves the purpose of formatting a web page.   Grid and Flexbox make using CSS3 a lot more straight forward and adds to the overall power of current web technologies.   Again as there are CSS libraries online it is possible to use one of these pre coded libraries then add to the functionality by using a custom css file.    An example of this is that this page uses "superhero.min.css" as the base library,  then the "layout.css" to make the changes to the format as required.   Because CSS files are prioritised on position the "layout.css" overrides the "superhero.min.css" should the settings exist in that file.

Rather than Use languages such as Javascript (JS) I would prefer to use a professional language that is compiled such as C#.   My reasoning is because JS can be disabled programmatically and as a result prevent hackers from exploiting JS vulnerabilities keeping the site more secure also the compiled code is not readable so is more secure.   Also JS has a performance hit so slows web sites down,  As modern Browsers also support web assemblies it seems to be the way forward.

This is a common language for current web sites it is Object Oriented and easy to learn so essentially is easy to translate into a language such as C#.   Having said that it is worth noting that the method of coding has changed over the years,  for example in times gone by programmers would try and make code complex in order to "show-off" their programming prowess.   The impact of this is that legacy code may use more code than required and as a result be less efficient, so in my opinion it is always worth refactoring code as this should not change the functionality but can impact the performance dramatically whether this is the file size of the code,  the performance or even both.   This is something I can do.
