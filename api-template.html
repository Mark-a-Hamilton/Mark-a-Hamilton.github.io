The purpose of the API Template is to provide a starting point when creating a new API the default Minimal API really needs some configuration in order to arrive at a meaningful starting point so the only thing to do is to concentrate on the meat of the API. The API-Template uses a restructured minimal API base, the default API is configured to allow it to connect to a SQL Server Database Log table, Endpoints provided, As swagger / OpenAPI is used by default and enables testing and documentation of the API.

Carrying on from the Overview there are many reasons why I have wrote an API Template so lets just list them here

  1. Understanding - As I wanted to Understand the intricacies of API's, previously I had only tested existing API's.
  2. Starting Point - On gaining an understanding I soon realised that there was quite a bit of configuration in order to get to a point where I could start coding to access the Database Table, Also known as the starting point. The reason why this is important is because if developers waste time by repeating necessary steps thus making then less efficient whereby a template will ensure that the developers only have to concentrate on the functionality of yjr Minimal API.

  4. Structure -
  5. Useage - Feel Free to Clone the API Template repository as it has the standard MIT License also it may be advantageous to adapt it based on your typical Use Case/
